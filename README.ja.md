# HistoryDiff 🌍✨

> **国や地域による歴史の「記述の差」を、テキスト比較（Diff）によって視覚的に解明する。**

同じ歴史的事象であっても、教科書に記載される内容や公式な見解は、国や地域によって大きく異なります。**HistoryDiff** は、その「認識の差」をテキストの比較（Diff）を通じて浮き彫りにし、背景の解説や検証ノート、参考文献を示すための双方向的な教育プラットフォームです。

## 🚀 主な機能

*   **テキスト差分ビュアー（Side-by-Side Diff）:** 異なる国や地域の教科書の記述や公式見解をリアルタイムで比較し、記述の「追加」「削除」「表現の違い」を単語レベルで視覚的に強調します。
*   **多言語ローカリゼーション対応:** Next.js の言語別サブルート（`/[lang]`）を利用し、**日本語 (`ja`)**、**英語 (`en`)**、**中国語 (`zh`)**、**韓国語 (`ko`)** の4言語で完全にローカライズされた体験を提供します。
*   **検証ノート（コミュニティノート）:** 特定の歴史的主張をファクトチェックし、中立的な「判定」、背景の歴史的文脈、および政府公式文書や学術論文といった信頼できる参考文献を示します。
*   **多角的なイベント検索・フィルタリング:** 歴史的カテゴリ（例：領土問題・主権、戦争・紛争、近現代の論争）、年代、視点をもつ国、フリーワードなどで瞬時にイベントを絞り込むことができます。
*   **自動翻訳パイプライン:** 日本語で記述されたコンテンツ（`-ja.md`、`notes.json`）をソースとし、Google翻訳APIをバックエンドとしたPythonスクリプト（`translate.py`、`translate_ko.py`）により、英語・中国語・韓国語へ自動翻訳・同期する仕組みを備えています。

## 🛠️ 技術スタック

*   **フロントエンドフレームワーク:** Next.js 16.2 (React 19, TypeScript)
*   **スタイリングとデザイン:** CSS変数（Custom Properties）を用いたバニラCSS。グラスモルフィズム、レスポンシブグリッド、モダンな色彩設計を組み合わせた洗練されたUI。
*   **差分比較エンジン:** `react-diff-viewer-continued`（構文レベル・文字レベルでの動的差分描画）
*   **マークダウン処理:** `react-markdown` および `gray-matter`（フロントマターのメタデータ解析とマークダウンテキストのレンダリング）
*   **アイコン:** `lucide-react` によるインタラクティブで軽量なアイコンデザイン。

## 📚 コンテンツデータ構造

各国の視点を記載するマークダウンファイルは、以下のようにYAMLフロントマターでメタデータが定義されています。

```markdown
---
id: "takeshima"
title: "竹島（独島）の領有権に関する記述"
category: "領土問題・主権"
year: "17世紀-現在"
location: "日本海"
country: "日本"
language: "ja"
source: "日本の外務省見解・教科書（一般的記述）"
---

ここから歴史的記述の本文を記載します。
通常のマークダウン記法が使用できます。
```

### 検証ノートの構造 (`notes.json`)

主張に対する検証内容と出典は、以下のようなJSON形式で管理されています。

```json
{
  "eventId": "takeshima",
  "notes": [
    {
      "id": "takeshima-claim-1",
      "claim": "検証対象となる主張・記述",
      "context": "なぜこれが論争になっているのかを示す背景解説",
      "verdict": "中立的な歴史的見地からのファクトチェック判定・総括",
      "sources": [
        {
          "title": "資料・論文・記事タイトル",
          "url": "https://example.com/source",
          "publisher": "発行機関・出版社・省庁など",
          "type": "academic"
        }
      ]
    }
  ]
}
```

## 🤖 翻訳自動化パイプライン

コンテンツ管理をシンプルにするため、コンテンツおよび検証ノートは**日本語をマスターソース**として作成し、他の言語へは自動翻訳スクリプトを用いて同期します。

### スクリプトの特長:
*   **賢いフロントマターの処理:** YAMLヘッダーの属性を番号付きリストにして一括翻訳することで、構造を壊さずに高速で翻訳します。不整合が生じた場合は、項目別の翻訳へと自動フォールバックします。
*   **マークダウン記号の保護:** マークダウンの記法やリンク、改行などのコード構造を維持したまま、自然な翻訳結果を適用します。
*   **高い信頼性:** APIの通信エラーに備え、指数バックオフを伴う最大5回の再試行処理（リトライロジック）を搭載しています。

### 実行方法:
プロジェクトのルートディレクトリで実行します:

```bash
# 日本語コンテンツを英語および中国語へ自動翻訳
python3 translate.py

# 日本語コンテンツを韓国語へ自動翻訳 (対象イベントのみ)
python3 translate_ko.py
```

## 💻 セットアップと起動方法

### 1. 前提条件
- **Node.js**: `v18.0.0` 以上
- **パッケージマネージャー**: npm / pnpm / yarn / bun
- **Python 3**: (オプション: 翻訳スクリプトを使用する場合のみ必要)

### 2. 依存関係のインストール
レポジトリをクローンまたは移動し、以下を実行します:

```bash
npm install
```

### 3. 開発サーバーの起動
ローカルで開発サーバーを起動します:

```bash
npm run dev
```

ブラウザで [http://localhost:3000](http://localhost:3000) を開くと、動作を確認できます。

### 4. 本番用ビルド
コードを最適化して本番用パッケージをビルドします:

```bash
npm run build
```
