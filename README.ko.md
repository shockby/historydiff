# HistoryDiff 🌍✨

> **국가와 지역에 따른 역사 기술의 차이점("기술 차이")을 텍스트 비교(Diff)를 통해 시각적으로 밝힙니다.**

동일한 역사적 사건이라 할지라도, 교과서에 기술된 내용이나 공식적인 역사 서술은 국가와 지역에 따라 크게 다릅니다. **HistoryDiff**는 이러한 인식의 차이를 텍스트 비교(Diff) 뷰어를 통해 직관적으로 비교하고, 풍부한 배경 분석, 검증 노트(Community Notes) 및 신뢰할 수 있는 학술/공식 참고 출처를 함께 제공하는 양방향 교육용 플랫폼입니다.

## 🚀 주요 기능

*   **나란히 비교하는 차이점 뷰어(Side-by-Side Diff):** 국가 및 지역별 역사 교과서 발췌록 또는 공식 역사적 서술을 실시간으로 나란히 비교하여 추가, 삭제, 어휘 및 표현 방식의 차이를 단어 단위로 시각화해 줍니다.
*   **다국어 로컬라이제이션 지원:** Next.js의 언어별 하위 라우트(`/[lang]`) 방식을 도입하여 **한국어 (`ko`)**, **영어 (`en`)**, **일본어 (`ja`)**, **중국어 (`zh`)**의 4개 국어로 완전하게 최적화된 사용자 경험을 제공합니다.
*   **커뮤니티 노트 및 사실 검증:** 특정 역사적 주장을 분석하고 이에 대해 중립적인 '판정', 역사적 배경 맥락 정보, 학술 논문이나 공식 기록물과 같은 공신력 있는 참고 서적을 제시하는 인터랙티브 카드 시스템입니다.
*   **다차원적인 이벤트 아카이브 검색:** 역사 카테고리(예: 영토 문제·주권, 전쟁·분쟁, 근현대사 논쟁), 역사적 연대, 서술 관점을 지닌 국가 또는 키워드 쿼리를 바탕으로 사건을 빠르게 필터링할 수 있습니다.
*   **자동 번역 파이프라인:** 일본어 원본 콘텐츠 및 검증 노트( 각각 `-ja.md`, `notes.json`)를 소스로 설정하고, Google 번역 API를 기반으로 제작된 커스텀 Python 스크립트(`translate.py`, `translate_ko.py`)를 통해 영어, 중국어, 한국어로 자동 번역 및 동기화합니다.

## 🛠️ 기술 스택

*   **프론트엔드 프레임워크:** Next.js 16.2 (React 19, TypeScript)
*   **디자인 및 스타일링:** CSS 변수(Custom Properties)를 적용한 바닐라 CSS, 글래스모피즘(Glassmorphism) 스타일 효과, 반응형 그리드 및 모던한 다크/라이트 테마 컬러 스키마.
*   **텍스트 비교 엔진:** `react-diff-viewer-continued`를 적용하여 고도로 튜닝된 문자 및 행 단위 다이내믹 디프(Diff) 렌더링.
*   **마크다운 파서:** `react-markdown` 및 `gray-matter`를 활용해 YAML 프론트매터(Frontmatter) 메타데이터 추출 및 서식 있는 본문 서술 렌더링.
*   **아이콘 라이브러리:** UI 직관성과 시각적 생동감을 더하는 반응형 `lucide-react` 아이콘 팩.

## 📚 데이터 스키마 규칙

개별적인 국가의 역사 기술 서술 파일은 YAML 형식의 헤더 블록(Frontmatter)을 갖는 표준 마크다운 문서로 작성됩니다.

```markdown
---
id: "takeshima"
title: "독도(다케시마) 영유권 서술 비교"
category: "영토 문제·주권"
year: "17세기-현재"
location: "동해 (일본해)"
country: "한국"
language: "ko"
source: "대한민국 외교부 영토 관련 백서 / 교과서 서술 인용"
---

여기에 구체적인 역사적 기술 및 발췌 문장을 기입합니다.
표준 마크다운 문법을 그대로 사용할 수 있습니다.
```

### 사실 검증 정보 데이터 형식 (`notes.json`)

특정 논쟁에 관한 사실적인 배경과 대조 데이터는 JSON 구조로 보관됩니다.

```json
{
  "eventId": "takeshima",
  "notes": [
    {
      "id": "takeshima-claim-1",
      "claim": "사실 검증의 대상이 되는 핵심 서술이나 주장",
      "context": "해당 진술이 어떤 맥락에서 역사적으로 쟁점이 되는지에 관한 배경 정보",
      "verdict": "중립적이고 역사 학술적 합의에 기초한 팩트체크 판정 결과 및 요약",
      "sources": [
        {
          "title": "참고 도서 또는 사료 명칭",
          "url": "https://example.com/source",
          "publisher": "출판사, 정부 부처 또는 학회 포털",
          "type": "academic"
        }
      ]
    }
  ]
}
```

## 🤖 번역 파이프라인 자동화

다국어 데이터 유지를 단순화하기 위해 **일본어 데이터를 모든 역사 정보의 표준 원본(Master Source)**으로 활용합니다. 함께 수록된 Python 번역 도구를 이용하면 변경 사항이 발생할 때마다 빠르고 편리하게 영어, 중국어, 한국어 파일로 자동 빌드 및 갱신할 수 있습니다.

### 번역 스크립트의 장점:
*   **스마트한 메타데이터 번역:** YAML 파싱 오류를 최소화하기 위해 프론트매터 키값들을 번호가 매겨진 리스트로 통합하여 일괄 번역 처리를 진행하며, 번역 어긋남이 발생할 시 개별 필드 번역 모드로 안전하게 회귀(Fallback)합니다.
*   **마크다운 마크업 보존:** 텍스트 내부의 특수 기호, 하이퍼링크 구조, 코드 블록을 손상시키지 않고 원활하게 번역합니다.
*   **지수 백오프 기반 재시도:** 구글 번역 모듈 통신 실패 혹은 네트워크 불안정 발생 시 최대 5회 동안 간격을 넓혀가며 지속적으로 재시도(Retry)하여 완성도를 보장합니다.

### 실행 방법:
로컬 터미널에서 프로젝트의 루트 폴더로 이동한 후 실행합니다.

```bash
# 일본어 원본 소스를 영어와 중국어 버전으로 번역 및 생성
python3 translate.py

# 일본어 원본 소스를 한국어 버전으로 번역 및 생성 (특정 이벤트 대상)
python3 translate_ko.py
```

## 💻 로컬 개발 환경 실행 방법

### 1. 사전 준비 환경
- **Node.js**: `v18.0.0` 또는 그 이상의 LTS 버전
- **패키지 매니저**: npm / pnpm / yarn / bun 중 선택 가능
- **Python 3**: (선택 사항: 데이터베이스를 추가하고 자동 번역 도구를 구동할 때에만 활용)

### 2. 패키지 설치
레포지토리를 복사한 뒤 다음 명령어로 의존 모듈을 설치합니다.

```bash
npm install
```

### 3. 개발 서버 구동
서버를 실행하고 변경 사항을 핫 리로딩으로 즉시 모니터링합니다.

```bash
npm run dev
```

터미널에 표시된 개발 URL(기본 포트 [http://localhost:3000](http://localhost:3000))을 인터넷 브라우저로 엽니다.

### 4. 프로덕션 패키지 빌드
배포를 위해 최적화된 파일들로 프로젝트를 프로덕션 빌드합니다.

```bash
npm run build
```

## 🤝 기여 안내 및 중립성 준수 규정

HistoryDiff는 비판적 사고, 미디어 리터러시, 세계사적 안목을 넓히기 위한 순수 교육용 플랫폼입니다. 특정 국가나 관점의 주장을 비호하거나 정치적인 스탠스를 대변하지 않으며, **"동일한 역사적 사실이 입장과 맥락에 따라 얼마나 다르게 표현되고 전달될 수 있는가"**를 있는 그대로 객관적으로 전개하는 데 집중합니다.

새로운 사실 검증 기록의 보강, 표현의 적확한 수정, 양질의 학술 서적 추천과 같은 전 세계 커뮤니티의 건설적인 기여를 따뜻하게 기다립니다!
