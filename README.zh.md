# HistoryDiff 🌍✨

> **通过文本对比（Diff）直观呈现不同国家和地区对同一历史事件的“记述差异”。**

即使是同一个历史事件，不同国家或地区的教科书内容以及官方叙事也会有很大差异。**HistoryDiff** 是一个交互式教育平台，旨在通过并排文本对比（Diff）凸显这些认知上的差异，并提供详细的背景解析、验证笔记以及学术/官方参考文献。

## 🚀 主要功能

*   **并排文本差异对比（Side-by-Side Diff）：** 实时对比不同国家/地区的教科书摘录或官方历史叙事，直观地在单词级别高亮显示“添加”、“删除”和“措辞差异”。
*   **多语言国际化支持：** 利用 Next.js 的本地化子路由（`/[lang]`），提供 **中文 (`zh`)**、**英文 (`en`)**、**日文 (`ja`)** 和 **韩文 (`ko`)** 四种语言的完整本地化体验。
*   **社区笔记与事实核查（Community Notes）：** 提供交互式卡片对特定的历史观点或主张进行事实核查，并给出中立的“结论”、历史背景分析以及政府公文、学术论文等权威参考文献。
*   **多维度事件检索与过滤：** 允许用户按历史类别（例如：领土主权、战争冲突、近现代争议）、历史年代、视角国家或自由文本关键词快速筛选事件。
*   **自动化内容翻译工作流：** 包含一套自定义翻译脚本（`translate.py`、`translate_ko.py`），以日语源内容（`-ja.md`、`notes.json`）为基准，使用 Google 翻译 API 自动翻译并同步至英语、中文和韩语。

## 🛠️ 技术栈

*   **前端框架：** Next.js 16.2 (React 19, TypeScript)
*   **样式与设计：** 原生 Vanilla CSS（基于 CSS 变量定义的设计系统），结合磨砂玻璃效果（Glassmorphism）、响应式网格布局以及现代化配色。
*   **差异对比引擎：** `react-diff-viewer-continued` 用于生成动态的行级/字符级差异对比。
*   **Markdown 解析：** 使用 `react-markdown` 和 `gray-matter` 解析 YAML 前言（Frontmatter）元数据并渲染富文本内容。
*   **图标库：** `lucide-react` 提供简洁、响应式的现代化矢量图标。

## 📚 内容数据结构

每个视角的文本文件都是一个包含 YAML 前言（Frontmatter）的 Markdown 文件，用以声明本地化的元数据属性：

```markdown
---
id: "takeshima"
title: "关于独岛（竹岛）主权记述争议"
category: "领土与主权"
year: "17世纪-至今"
location: "日本海（东海）"
country: "日本"
language: "zh"
source: "日本外务省官方见解 / 教科书一般记述"
---

此处为具体的历史记述正文。
支持标准的 Markdown 格式。
```

### 事实核查笔记数据结构 (`notes.json`)

笔记以结构化形式管理事实核查、背景和证据来源：

```json
{
  "eventId": "takeshima",
  "notes": [
    {
      "id": "takeshima-claim-1",
      "claim": "需要被核查的具体历史主张",
      "context": "该主张为何存在争议的历史背景分析",
      "verdict": "基于中立历史视角的ファクトチェック（事实核查）结论",
      "sources": [
        {
          "title": "文献、论文或档案标题",
          "url": "https://example.com/source",
          "publisher": "发布机构、学报或政府部门",
          "type": "academic"
        }
      ]
    }
  ]
}
```

## 🤖 自动化内容翻译工作流

为了简化内容维护工作，所有内容及核查笔记均以**日语作为原始主数据（Master Source）**进行编写，然后运行 Python 脚本自动翻译同步至其他语言版本。

### 翻译脚本特色：
*   **智能前言翻译：** 将 YAML 标头字段重组为结构化编号列表进行批量翻译，最大程度保留属性结构，并在格式解析不一致时自动退回至逐字段单译模式。
*   **完备的 Markdown 保护：** 在翻译正文时，完美保护 Markdown 语法标记、超链接以及代码行结构。
*   **重试与高可靠性：** 针对 API 请求限制，支持带有指数退避机制的最多 5 次失败重试。

### 运行方式：
在项目根目录下，使用终端运行以下命令：

```bash
# 将日语源文件自动翻译为英语和中文版本
python3 translate.py

# 将日语源文件自动翻译为韩语版本 (针对特定事件)
python3 translate_ko.py
```

## 💻 本地开发指南

### 1. 前置条件
- **Node.js**: `v18.0.0` 或更高版本
- **包管理器**: npm / pnpm / yarn / bun
- **Python 3**（可选，仅在更新内容数据库运行翻译时需要）

### 2. 安装依赖
克隆或进入项目目录，执行以下命令安装前端依赖：

```bash
npm install
```

### 3. 运行本地开发服务器
启动本地热更新开发服务：

```bash
npm run dev
```

在浏览器中打开 [http://localhost:3000](http://localhost:3000) 即可实时查看应用。

### 4. 生成生产构建
在部署前生成高度优化的静态和服务器渲染生产包：

```bash
npm run build
```
